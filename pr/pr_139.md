## Changes Implemented

- Replaced in-memory sorting by upvotes with database-level `ORDER BY` using `Sequelize.literal('COALESCE(array_length("upvotes", 1), 0)')`
- Applied `LIMIT` and `OFFSET` directly in the SQL query instead of fetching all rows and slicing in JavaScript
- Changed from `Feedback.findAll()` + manual pagination to single paginated query

## Technical Details

### Backend

- **`backend/controllers/communityController.js`**: The `getFeedbackList` function now sorts by upvote array length at the database level and applies limit/offset directly, so only the requested page's rows are loaded into memory.
