## Changes Implemented

- Rewrote `deleteExam` to cascade-delete all associated StudyPlans, PYQs, Quizzes, QuizAttempts, Notes, Flashcards, Progress records, Topics, and Subjects
- Rewrote `deleteSubject` to cascade-delete all associated Topics, PYQs, Quizzes, QuizAttempts, Notes, Flashcards, and Progress records
- Rewrote `deleteTopic` to cascade-delete all associated Quizzes, QuizAttempts, Flashcards, and Progress records, and nullify the topic reference on Notes
- Wrapped all cascade deletions in database transactions to ensure atomicity

## Technical Details

### Backend

- **`backend/controllers/academicController.js`**: All three DELETE handlers (`deleteExam`, `deleteSubject`, `deleteTopic`) now explicitly destroy all child/orphaned records before removing the target entity. Each operation runs inside a Sequelize transaction so that a failure mid-way rolls back all changes.

### Database

Previously, deleting an Exam only removed its immediate Subjects and Topics, leaving orphaned rows in StudyPlan, PYQ, Quiz, QuizAttempt, Note, Flashcard, and Progress tables. The same issue existed for Subject and Topic deletions. Now every known dependent record is cleaned up in the correct order (children before parents) to respect any existing foreign key relationships.
