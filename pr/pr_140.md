## Changes Implemented

- Changed `getDashboardStats` and `getStudyHours` to filter progress records by `updatedAt >= (NOW() - 6 days)` instead of `limit: 7`
- Added aggregation by calendar date using `fn('DATE', col('updatedAt'))` so each day appears exactly once
- Filled in missing days with zero study hours/completion so the chart always shows a complete 7-day window
- Removed hardcoded demo data fallback — brand-new users see zeros instead of fake data

## Technical Details

### Backend

- **`backend/controllers/progressController.js`**: Both `getDashboardStats` and `getStudyHours` now use a date-filtered query with GROUP BY DATE and fill in gaps client-side by iterating over the last 7 calendar days.
